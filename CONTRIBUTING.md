# Contributing to Decode

We love your input! We want to make contributing to Decode as easy and transparent as possible.

## Development Process

1. Fork the repository and create your branch from `main`.
2. If you've added code, add tests if applicable.
3. Ensure the code builds and lints cleanly.
4. Update the documentation if needed.
5. Issue a pull request.

## Code Style

- Follow Kotlin coding conventions
- Use Jetpack Compose for all UI components
- Use Material 3 design patterns
- Keep functions focused and small
- Use meaningful variable and function names
- Add KDoc comments for public APIs

## Pull Request Process

1. Update the README.md with details of changes if needed.
2. Update the version number in build.gradle.kts if releasing.
3. Ensure any install or build dependencies are up to date.
4. The PR will be merged once you have the sign-off of the maintainer.

## Report bugs using GitHub Issues

We use GitHub Issues to track bugs. Report a bug by [opening a new issue](https://github.com/Llucs/Decode/issues/new).

## Write bug reports with detail, background, and sample code

**Great Bug Reports** tend to have:

- A quick summary and/or background
- Steps to reproduce (be specific: what device, what OS)
- What you expected would happen
- What actually happens
- Notes (possibly including why you think this might be happening, or stuff you tried that didn't work)
